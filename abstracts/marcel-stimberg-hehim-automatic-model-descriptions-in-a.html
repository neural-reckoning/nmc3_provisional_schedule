Automatic model descriptions in a neural network simulator

Marcel Stimberg (he/him)

Vigneswaran C, Indian Institute of Technology Madras, Chennai

Abstract

Computational neuroscience simulations are often written as scripts that make use of a neural simulator package. The details of a model implemented in such a script might not be obvious to a reader not familiar with the simulator. When a researcher presents such a model in a publication, they will therefore rather describe it with text and equations. Creating such a description, however, is error-prone, since it is up to the researcher to make sure that it accurately presents the simulated model. We therefore propose that it would be very useful if the simulator itself could generate a model description automatically. Here we present a prototype of such a system for the Brian simulator. It is built on the same framework that powers Brian's code generation process and is therefore guaranteed to give results that are consistent with the actual simulation. The new model description framework provides a basic exporter that generates a machine-readable representation which abstracts away all the details of Brian's internal representations and takes care of more subtle issues such as the meaning of referred constants and the order of variable assignments. This exporter is meant to be used as a base for other types of exporters. As a proof of concept, we show an export to the Markdown format, providing a human-readable description of a model. Such a description can be useful to exchange with other researchers that are not familiar with the Brian simulator, as a basis for a model description shared as part of a publication or website, or simply as a means for debugging a model. Finally, we discuss the limitations of such descriptions and the possibility of annotating it with additional semantic information derived from the model.
